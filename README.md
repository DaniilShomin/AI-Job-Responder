# AI Job Responder

Автоматический инструмент для отклика на вакансии с помощью ИИ. Поддерживает поиск и отклик на вакансии на платформах:

- [hh.ru](https://hh.ru)
- [career.habr.com](https://career.habr.com)

Скрипт собирает вакансии, фильтрует их релевантность через AI, генерирует уникальное сопроводительное письмо и отправляет отклик через браузер.

## Возможности

- 🔍 **Сбор вакансий** — парсинг вакансий с поисковой выдачи
- 🤖 **AI-фильтрация** — автоматическая проверка релевантности вакансии по описанию
- ✍️ **Генерация откликов** — уникальные сопроводительные письма на основе вашего резюме
- 🌐 **Мультиплатформенность** — поддержка hh.ru и career.habr.com
- 📝 **История обработки** — сохранение обработанных URL, чтобы не откликаться повторно
- 🔒 **Сохранение сессии** — авторизация сохраняется между запусками

## Требования

- Python `3.13+`
- [uv](https://docs.astral.sh/uv/) (рекомендуется)
- Chromium (для Playwright)
- API-ключ для AI-провайдера (OpenRouter или совместимый endpoint)

## Установка

1. Клонируйте репозиторий:

```bash
git clone <repo-url>
cd <project-folder>
```

2. Установите зависимости:

```bash
uv sync
```

3. Установите браузер для Playwright:

```bash
uv run playwright install chromium
```

## Настройка

### 1. Переменные окружения

Создайте файл `.env` в корне проекта:

```env
# Обязательно
API_KEY=your_api_key_here

# AI-провайдер (опционально)
API_BASE_URL=https://routerai.ru/api/v1
MODEL=deepseek/deepseek-v4-flash

# Платформы (укажите хотя бы одну)
HH_SEARCH_URL=https://hh.ru/search/vacancy?text=python&area=1
HABR_SEARCH_URL=https://career.habr.com/vacancies?q=python

# Браузер
HEADLESS=false

# Таймауты
TIMEOUT=30
LOGIN_TIMEOUT=60000

# Файлы данных
DATA_FILE=data.json
RESUME_FILE=resume.txt
PROMPT_FILE=prompt.txt

# Лимит откликов на одну платформу за запуск (по умолчанию: 10)
RESPONSE_LIMIT_PER_PLATFORM=5
```

**Описание переменных:**

| Переменная | Обязательная | Описание | По умолчанию |
|------------|-------------|----------|--------------|
| `API_KEY` | Да | Ключ доступа к AI API | — |
| `API_BASE_URL` | Нет | Базовый URL API | `https://routerai.ru/api/v1` |
| `MODEL` | Нет | Модель для генерации | `stepfun/step-3.5-flash:free` |
| `HH_SEARCH_URL` | Нет | Прямая ссылка на выдачу hh.ru | — |
| `HABR_SEARCH_URL` | Нет | Прямая ссылка на выдачу career.habr.com | — |
| `HEADLESS` | Нет | Режим браузера (`true`/`false`) | `true` |
| `TIMEOUT` | Нет | Таймаут AI-запроса (сек) | `10` |
| `LOGIN_TIMEOUT` | Нет | Таймаут ожидания входа (мс) | `60000` |
| `DATA_FILE` | Нет | Файл с обработанными URL | `data.json` |
| `RESUME_FILE` | Нет | Путь к резюме | `resume.txt` |
| `PROMPT_FILE` | Нет | Путь к шаблону отклика | `prompt.txt` |
| `RESPONSE_LIMIT_PER_PLATFORM` | Нет | Лимит откликов за запуск | `10` |

### 2. Файл резюме

Создайте `resume.txt` с текстом вашего резюме. Этот текст будет использоваться AI для генерации релевантных откликов.

**Пример:**

```text
Иван Иванов
Python-разработчик с 3 годами опыта.

Навыки:
- Python, Django, FastAPI
- PostgreSQL, Redis
- Docker, Git
- REST API, GraphQL

Опыт работы:
- Разработка микросервисов
- Оптимизация SQL-запросов
- Код-ревью и менторство

Контакты:
- Email: ivan@example.com
- Telegram: @ivanov
```

### 3. Файл промпта

Создайте `prompt.txt` с инструкциями для AI по составлению отклика.

**Пример:**

```text
Ответ должен быть на русском языке. Содержание должно быть уникальным, избегай шаблонных фраз.

Структура ответа:
1. Приветствие
2. Краткое представление (опыт, навыки)
3. Почему интересна эта вакансия и компания
4. Как ваш опыт соответствует требованиям
5. Готовность к собеседованию и благодарность
6. Контактная информация

Контакты:
Email: ivan@example.com
Telegram: @ivanov
GitHub: github.com/ivanov
```

## Запуск

```bash
uv run python main.py
```

### Что происходит при запуске

1. Инициализация AI-клиента и загрузка локальных файлов (`resume.txt`, `prompt.txt`)
2. Открытие браузера и попытка восстановить сессию из `auth_state.json`
3. Переход на страницу поиска вакансий (hh.ru, career.habr.com)
4. Если сессия неактивна — ожидание ручного входа в аккаунт
5. Для каждой вакансии:
   - Считывание заголовка и описания
   - AI-анализ релевантности профессии
   - Генерация сопроводительного письма
   - Отправка отклика
   - Сохранение URL в историю
6. Переход на следующую страницу результатов
7. Повтор до достижения лимита откликов или исчерпания результатов

## Troubleshooting

### Браузер не запускается / ошибки Playwright

```bash
uv run playwright install chromium
```

Проверьте, что система поддерживает запуск Chromium (для headless-режима достаточно минимальных зависимостей).

### Не удаётся войти в аккаунт

- Запустите с `HEADLESS=false`, чтобы видеть окно браузера
- Проверьте, не требуется ли CAPTCHA или 2FA
- Удалите `auth_state.json` и выполните вход заново

### Ошибка загрузки `resume.txt` или `prompt.txt`

- Убедитесь, что файлы существуют в корне проекта
- Проверьте пути в `.env` (переменные `RESUME_FILE`, `PROMPT_FILE`)

### Повреждён `data.json`

```bash
echo '[]' > data.json
```

### Таймауты AI API

- Увеличьте `TIMEOUT` в `.env` (например, до `30` или `60`)
- Проверьте корректность `API_KEY` и доступность сервиса

### Браузер закрывается при работе

- Проверьте наличие свободной оперативной памяти
- Запустите в видимом режиме (`HEADLESS=false`) для диагностики
- Убедитесь, что нет конфликтов с другими запущенными браузерами

## Важные замечания

- Скрипт зависит от текущей вёрстки сайтов. При изменениях UI возможны сбои.
- Обработка вакансий выполняется последовательно (не параллельно).
- Качество откликов напрямую зависит от качества резюме и промпта.
- Скрипт пропускает вакансии, требующие ответов на дополнительные вопросы работодателя.
