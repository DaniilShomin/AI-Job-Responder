## HH Auto Responder

Скрипт для автоматизации откликов на вакансии на `hh.ru`:
- собирает вакансии со страницы поиска;
- фильтрует их через AI по релевантности профессии;
- генерирует сопроводительное письмо;
- отправляет отклик через Playwright;
- сохраняет обработанные URL в `data.json`.

## Требования

- Python `3.13+`
- `uv` (рекомендуется для установки и запуска)
- Установленный браузер Chromium для Playwright
- API-ключ для OpenRouter (или совместимого endpoint, указанного в коде)

## Установка

1. Клонируйте репозиторий и перейдите в папку проекта.
2. Установите зависимости:

```bash
uv sync
```

3. Установите браузер для Playwright:

```bash
uv run playwright install chromium
```

4. Создайте `.env` в корне проекта.
5. Подготовьте файлы:
- `resume.txt` — ваш текст резюме;
- `prompt.txt` — шаблон/пример отклика;
- `data.json` — файл со списком уже обработанных вакансий (можно не создавать заранее).

## ENV-переменные

Читаются из `.env` (см. `app/config.py`).

Обязательная:
- `API_KEY` — ключ доступа к AI API.

Опциональные:
- `MODEL` — модель для генерации (по умолчанию: `stepfun/step-3.5-flash:free`)
- `HH_SEARCH_URL` — прямая ссылка на готовую выдачу вакансий (если не указано, используется переход через UI)
- `HEADLESS` — режим браузера `true/false` (по умолчанию: `False`)
- `TIMEOUT` — таймаут AI-запроса в секундах (по умолчанию: `10`)
- `DATA_FILE` — путь к JSON с обработанными URL (по умолчанию: `data.json`)
- `RESUME_FILE` — путь к файлу резюме (по умолчанию: `resume.txt`)
- `PROMPT_FILE` — путь к файлу шаблона отклика (по умолчанию: `prompt.txt`)

Пример `.env`:

```env
API_KEY=your_openrouter_api_key
MODEL=stepfun/step-3.5-flash:free
HH_SEARCH_URL=https://hh.ru/search/vacancy?...
HEADLESS=false
TIMEOUT=10
DATA_FILE=data.json
RESUME_FILE=resume.txt
PROMPT_FILE=prompt.txt
```

## Запуск

```bash
uv run python main.py
```

Что происходит при запуске:
1. Инициализируется AI-клиент и загрузка локальных файлов.
2. Открывается браузер, выполняется вход на `hh.ru` (или используется сохранённая сессия).
3. Открывается выдача вакансий.
4. Для каждой вакансии:
   - читается описание;
   - определяется релевантность;
   - генерируется текст отклика;
   - отправляется отклик.
5. Список обработанных ссылок сохраняется в `data.json`.

## Ограничения

- Скрипт зависит от текущей вёрстки и текстов UI на `hh.ru`; при изменениях селекторов возможны сбои.
- Используется синхронный Playwright и последовательная обработка вакансий.
- AI-классификация релевантности и генерация отклика зависят от качества ответа модели.
- Требуется корректный логин/сессия на `hh.ru`; состояние сохраняется в `auth_state.json`.

## Troubleshooting

### 1) `API_KEY не установлен в переменных окружения`
- Проверьте наличие `.env` в корне проекта.
- Убедитесь, что в нём задан `API_KEY`.

### 2) Ошибки Playwright / браузер не запускается
- Выполните установку браузера:
  - `uv run playwright install chromium`
- Проверьте, что окружение поддерживает запуск Chromium.

### 3) Не удаётся войти в hh.ru
- Запустите скрипт в видимом режиме (по факту сейчас так и запускается).
- Проверьте ручной вход, CAPTCHA/2FA и актуальность сохранённого `auth_state.json`.
- При необходимости удалите `auth_state.json` и авторизуйтесь заново.

### 4) `Ошибка загрузки резюме` / `Ошибка загрузки шаблона`
- Проверьте существование и права доступа к `resume.txt` и `prompt.txt`.
- Проверьте пути `RESUME_FILE` и `PROMPT_FILE` в `.env`.

### 5) Ошибка чтения/записи `data.json`
- Проверьте права на директорию проекта.
- Проверьте, что `DATA_FILE` указывает на файл, а не на директорию.
- Если JSON повреждён, исправьте файл или создайте новый пустой список `[]`.

### 6) Таймауты или ошибки AI API
- Увеличьте `TIMEOUT` (например, `20`).
- Проверьте корректность `API_KEY`, лимиты и доступность API.
